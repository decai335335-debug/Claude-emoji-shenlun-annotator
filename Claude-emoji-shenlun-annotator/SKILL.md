# 申论深度标注 Skill

## 一、适用场景

- 用户给出**申论原题（题干 + 给定资料）**，要求生成"标注版 Markdown"。
- 用户要求按照《申论刷题第二期》20 天范本的格式继续标注新材料。
- 用户要求从已有的非标注文本中提取采分点并按统一样式标注。

## 二、文件整体结构（严格遵守）

输出必须为一份完整 Markdown，模板如下：

```markdown
# 📝 第X天：{考试类型} - {主题}

**题干**：{原题干文字}。({分值}分)

---

## 【给定资料标注版】

{段落1（已标注）}

---

{段落2（已标注）}

---

... 其他段落 ...

---

## 📊 {主题}标注汇总

| {维度} | {内容} | {数据/案例} |
|:---|:---|:---|
| **xxx** | xxx | xxx |

**核心经验**：xxx + xxx + xxx
```

关键点：
- 顶部 `# 📝` 一级标题。
- 题干用 `**题干**：` 前缀，末尾保留分值。
- 每个自然段用 `---` 分隔（前后空行）。
- 末尾**必须**生成 `## 📊 ...标注汇总` 表格 + "核心经验 / 核心启示 / 核心机制"一句话总结。

## 三、行内标注格式（最核心的规则）

### 1. 加粗

用 `**text**` 包裹**原材料术语、数字、机构名、政策名、人名+身份、核心动词短语**。
- ✅ `**农村农业局**派了**技术员点对点指导**`
- ✅ `**500多万株**`、`**40多万元**`
- ❌ 不要加粗长句、形容词、连接词。

### 2. 行内标注（括号 + emoji + 冒号）

**格式**：`` `【{emoji}{中/英标签}: {精炼内容}】` ``

- 整体包在**反引号**里（`` ` ` ``），保证在渲染时与正文区分。
- emoji **紧贴**标签词，之间不加空格。
- 冒号**后**加一个半角或全角空格均可，保持整篇一致。
- 标注**紧跟**被标注句末，**不换行**。
- 多个标注**连写**，不加分隔符：`` `【...】【...】【...】` ``

示例：

```
"市**农村农业局**派了**技术员点对点指导**我们。`【🔧Assistance measure 1: 技术下乡——农村农业局派技术员点对点指导】【💯Data: 500多万株、40万纯利润】`
```

### 3. 子项表达

标签内部若需展开多点，用 `——` 引出，用 `+` 或 `、` 连接：
- `【🔧Approach: 信用评价扩围+M级企业纳入+以信换贷】`
- `【🔧Measures: 改善微循环+贯通主动脉】`

### 4. 语言风格

**中英混标均可**，沿用范本习惯（20 篇中以英文标签 + 中文内容为主）：
- `【⭐Core viewpoint: ...】` / `【⭐核心观点：...】`
- `【🔧Approach 1: ...】` / `【🔧做法1：...】`

在同一份文件内**保持统一**；续写他人已有文件时沿用其风格。

## 四、Emoji 语义体系（记住 8 大高频标签）

| 语义 | emoji | 典型标签 | 用途 |
|:---|:---:|:---|:---|
| **核心观点** | ⭐ 💎 🏆 ✨ | Core viewpoint / Core experience / Core concept | 全段主旨、结论升华、点题句 |
| **措施做法** | 🔧 ✅ 🛠️ ⚙️ | Approach / Measure / Solution / Mechanism | 政府/企业具体做法，最常用 |
| **问题困境** | ⚠️ ❗ 🚨 | Problem / Challenge / Core problem | 负面现象、痛点、瓶颈 |
| **数据事实** | 💯 📊 📈 | Data / Key data / Achievement data | 数字、比例、规模、金额 |
| **成效反馈** | 🚀 🎉 | Effect / Achievement / Result | 做法之后的积极变化 |
| **背景政策** | 🌐 🏛️ 📍 | Background / Policy background | 非采分的时空/政策铺垫 |
| **概念定义** | 📖 💡 🧩 | Concept / Definition / Idea | 新名词、新提法解释 |
| **目标愿景** | 🏆 🎯 | Goal / Vision / Direction | "打造……""建设……""迈向……" |

次高频:
- `👥` 群众声音 / `💬` 原文引用 / `🔎` 调研分析 / `🏷️` 分类标签 / `💎` 亮点价值。

**一句话原则**：**做法→🔧、问题→⚠️、数据→💯、背景→🌐、结论/观点→⭐、成效→🚀、概念→📖**。

## 五、标注内容的选取标准

对每一段材料，按以下顺序识别要点：

1. **领导讲话、总结句、主旨句** → `⭐核心观点`
2. **具体动词短语（派、建、推行、打造、出台、联合、搭建）** → `🔧措施/做法`
3. **数字、百分比、金额、时间跨度** → `💯数据`
4. **"问题""瓶颈""短板""困难""……不足""……难"** → `⚠️问题`
5. **"有效……""提升……""实现……""解决了……"** → `🚀成效`
6. **政策文件、会议、历史阶段、地域背景** → `🌐背景`
7. **新概念引号内术语（"数字孪生""东数西算""柔的方法"）** → `📖概念`
8. **目标愿景（"打造……圈""建设……体系"）** → `🏆目标`

一句话多要点时 **连续堆标签**，不要合并。

## 六、末尾汇总表生成规则

1. 表格标题固定 `## 📊 {主题词}标注汇总` 或 `## 📊 ...核心标注`。
2. **列数 2–4 列**，首列用 `**加粗**` 类别名。
3. 对齐语法：`|:---|:---|` (左对齐) 或 `|:---:|` (居中，序号列)。
4. 表格之后补**一句话核心经验**，用 `+` 连接要素：
   - `**核心经验**：居民参与 + 多方共担 + 市场化运营`
   - `**核心启示**：守 = 保留特色；新 = 与时俱进`
   - `**核心机制**：政府牵头 + 企业参与 + 群众监督`

## 七、标注工作流（你的执行步骤）

1. **读题干** → 判定文体（概括题/对策题/分析题/应用文/大作文），题干决定要点颗粒度。
2. **分段处理**，每段：
   - (a) 通读识别主体、动作、数据、问题、成效；
   - (b) **加粗**原文术语；
   - (c) 在相应句末**追加**`` `【...】` ``标签，多标签连写；
   - (d) 段末用 `---` 分隔。
3. **不要改写正文**，只做"加粗 + 追加标签"两件事。引号内原话完整保留。
4. 全文处理完毕后 → 生成末尾汇总表 + 一句话核心经验。
5. 最终交付**完整 Markdown 文本**（不要用工具直接写文件，除非用户明确要求保存路径）。

## 八、常见错误与禁忌

- ❌ 把标签写成独立一行 → 必须紧跟句末。
- ❌ 忘记反引号包裹 → `` `【...】` `` 是硬性要求。
- ❌ 在标签里写长句解释 → 标签要**短语化、术语化**，≤ 20 字为宜。
- ❌ 整段堆一个超长标签 → 应拆分为 2–4 个语义独立的标签。
- ❌ 改写、压缩、翻译原文 → 只可加粗与追加。
- ❌ emoji 用错语义（用 🔧 标问题、用 ⚠️ 标数据）→ 参照第四节对照表。
- ❌ 中英风格在同一文件里混乱切换。
- ❌ 漏掉末尾 📊 汇总表。

## 九、风格速查（Cheatsheet）

```
**关键术语**{紧接原文}`【🔧Approach N: 短语化概括——子项1+子项2】【💯Data: 数字+量纲】`
```

```
问题句：…… **瓶颈** ……`【⚠️Problem N: 核心痛点】`
成效句：…… **提升/降低/实现** ……`【🚀Effect: 量化结果】`
观点句：……"X 市长说"`【⭐Core viewpoint: 点题金句浓缩】`
概念句："X" 是 ……`【📖Concept: 一句话定义】`
```

记住：**标注是为了答题找点**。每一个 🔧 / ⚠️ / 💯 都应可直接摘抄进答案要点。
